# Ремонт24 - Система учёта заявок на ремонт ПК и ноутбуков(документация будет изменяться в будущем)

## О проекте

**Ремонт24** - это полноценная информационная система для автоматизации учёта заявок на ремонт компьютерной техники. Проект разработан в рамках производственной практики и представляет собой рабочее fullstack-приложение с авторизацией, разделением ролей, управлением статусами заявок и историей изменений.

### Для чего нужен этот сервис?

Сервис позволяет:
- **Клиентам** - создавать заявки на ремонт, отслеживать статус выполнения, общаться с мастерами через комментарии.
- **Мастерам** - просматривать назначенные заявки, изменять статусы, добавлять заключения и использованные запчасти.
- **Администраторам** - управлять пользователями, назначать мастеров на заявки, контролировать весь процесс ремонта.

### Роли пользователей

| Роль       | Возможности                                                                 |
|------------|-----------------------------------------------------------------------------|
| **Клиент** | Создание заявки, просмотр своих заявок, добавление комментариев             |
| **Мастер** | Просмотр всех заявок, изменение статуса, добавление внутренних комментариев |
| **Админ**  | Полный доступ: управление пользователями, назначение мастеров, удаление     |

### Основной цикл заявки
Новая → Принята → Диагностика → Ремонт → Готова → Выдана

Отмена, комментарий, запчасти доступны на соответствующих этапах

## Технологии

### Backend
- **Python 3.13**
- **Flask** - микрофреймворк для API
- **Flask-SQLAlchemy** - ORM для работы с БД
- **Flask-JWT-Extended** - JWT-авторизация
- **Flask-CORS** - кросс-доменные запросы
- **PostgreSQL** - реляционная база данных
- **pgAdmin4** - управление БД

### Инфраструктура
- **Docker** и **Docker Compose** - контейнеризация
- **GitHub Actions** - CI/CD автоматизация
- **Render.com** - деплой

### Инструменты
- **Git** - контроль версий
- **.env** - управление переменными окружения

## Как запустить проект

### Требования

- Python 3.13
- PostgreSQL 15+ или Docker
- Git
- Docker и Docker Compose

### Локальный запуск (без Docker)

#### 1. Клонировать репозиторий

git clone https://github.com/suhrobabdusharipov/proizvodstvenyapractica.git  
cd proizvodstvenyapractica/backend  

#### 2. Создать виртуальное окружение

python -m venv venv  
source venv/bin/activate      # Linux/Mac  
venv\Scripts\activate         # Windows  

#### 3. Установить зависимости

pip install -r requirements.txt

#### 4. Настроить переменные окружения

Создайте файл .env в папке backend/:

DB_HOST=localhost  
DB_PORT=5432  
DB_NAME=pc_repair_db  
DB_USER=ваш_юзер  
DB_PASSWORD=ваш_пароль  

JWT_SECRET_KEY=секретный_ключ_минимум_32_символа  
SECRET_KEY=flask_secret_key  

#### 5. Создать базу данных в PostgreSQL

Подключитесь к PostgreSQL и выполните:

CREATE DATABASE pc_repair_db;

#### 6. Инициализировать базу данных

python init_db.py

#### 7. Запустить сервер

python main.py

Сервер будет доступен по адресу: http://localhost:5000

#### 8. Проверить работоспособность

curl http://localhost:5000/health

Ожидаемый ответ: {"status":"ok"}


### Запуск через Docker (позже будет добавлен)

#### 1. Убедитесь, что Docker установлен

docker --version  
docker-compose --version

#### 2. Клонировать репозиторий

git clone https://github.com/suhrobabdusharipov/proizvodstvenyapractica.git  
cd proizvodstvenyapractica

#### 3. Запустить все сервисы одной командой

docker compose up --build

Docker Compose поднимет:
- PostgreSQL (порт 5432)
- Backend Flask (порт 5000)
- pgAdmin4 (порт 5050)

#### 4. Инициализировать базу данных внутри контейнера

В отдельном терминале:

docker compose exec backend python init_db.py

#### 5. Открыть в браузере

- API: http://localhost:5000
- Health check: http://localhost:5000/health
- pgAdmin4: http://localhost:5050 (email: ваша_почта, пароль: ваш_пароль)

#### 6. Остановить все контейнеры

docker compose down
